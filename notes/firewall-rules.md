# Firewall Rules Notes

Firewall rules should be understandable, specific, and tied to business or lab requirements.

| Principle | Explanation |
|---|---|
| Default deny inbound | Block unsolicited inbound traffic unless explicitly required. |
| Least privilege | Allow only the ports, protocols, and sources needed. |
| Document purpose | Every rule should have a reason. |
| Review regularly | Old rules can become unnecessary exposure. |
| Validate changes | Confirm that allowed traffic works and blocked traffic is denied. |
